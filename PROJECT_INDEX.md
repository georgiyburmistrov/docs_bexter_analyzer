# PROJECT_INDEX

## Название проекта
Анализатор логов производственного оборудования

## Краткое описание
Проект по разработке MVP веб-приложения для загрузки, обработки, поиска и анализа логов производственного оборудования с графическим UI. MVP предназначен для быстрого поиска ошибки и восстановления полной хронологии одного производственного цикла по трём логам.

## Цель проекта
Создать приложение, которое позволит:
- быстро находить место возникновения ошибки без ручного просмотра логов целиком;
- формировать сквозную запись одного производственного цикла;
- показывать хронологию событий, ошибку и статус цикла;
- подготавливать данные для дальнейшего развития аналитики и интеграции с Grafana.

## Текущий статус проекта
- **Текущий этап:** Этап 2. Обследование логов и описание входных данных
- **Предыдущий завершённый этап:** Этап 1. Инициация проекта и подготовка ТЗ
- **Следующий ключевой результат:** описание логов `FADATA`, `FA`, `RFIDCore2`, выделение полей и кандидатов на корреляцию

## Границы MVP
### В MVP входит
- ручная загрузка 3 логов через главную страницу;
- поиск по тележке, материалу, началу и концу периода;
- парсинг, нормализация и корреляция трёх логов;
- формирование одной сквозной записи по одному производственному циклу;
- показ единого потока событий, ошибки, статуса и выдержки из исходного лога ошибки;
- показ технического блока результата обработки.

### В MVP не входит
- подключение к SCADA;
- автоматическая загрузка логов;
- выбор производственной машины;
- поддержка нескольких машин;
- отдельная страница журнала загрузок и обработок;
- интеграция в production-контур;
- Grafana как часть MVP.

## Входные логи MVP
- `FADATA` — интеграционный/протокольный лог FA ↔ PLC и отправки в FAM
- `FA` — прикладной лог приложения FA
- `RFIDCore2` — технический лог RFID-подсистемы

## Основной пользовательский сценарий MVP
1. Пользователь открывает главную страницу.
2. Загружает 3 файла логов.
3. Указывает один или несколько фильтров:
   - номер тележки;
   - материал;
   - начало периода;
   - конец периода.
4. Нажимает кнопку `Поиск`.
5. Система обрабатывает логи и отображает результат на отдельной странице.

- сначала приоритетно обследуется и описывается лог `FADATA`;
- сначала подготавливаются правила парсинга для `FADATA`;
- допускается, что первое рабочее ядро MVP может быть реализовано на основе `FADATA`;
- `FA` и `RFIDCore2` сохраняются как обязательные источники для последующего расширения и уточнения результата.
